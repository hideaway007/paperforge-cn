# Part 5 Formal Manuscript Authoring

你的任务是约束 Part 5 写正文的 LLM agent，把 `outputs/part5/manuscript_v1.md` 从一份“能审的草稿”直接写成接近正式中文论文的正文形态。`manuscript_v1.md` 仍是中间稿，不是 canonical final；但它的结构必须足够正式，才能让 Part 5 review 和 revise 基于真实论文形态工作。

只处理结构、标题层级、论文部件和表达收束；不得新增研究事实、source、citation、案例事实、图纸信息、数据或研究结论。

## 使用边界

- 研究内容只能来自 Part 1-4 canonical artifacts 与 Part 5 prep package。
- 真实论文只作为结构参照：抽取章节顺序、标题层级、摘要/关键词、章节小结、参考文献等论文形态。
- 不得迁移参考论文的具体主题、对象、地域、人物、项目、案例名、历史事实、专有概念或结论。
- 本 skill 不拥有 canonical artifact；文件写入权限仍由 `$part5-draft-manuscript` 或 runtime Part 5 draft owner 决定。
- 如果材料不足以生成某个正式部件，不得伪造；应保留待核对标记或交给 review/report。

## 与当前草稿问题的差异

若输入或 fallback 草稿类似报告型 Markdown，常见问题是：

- 顶部出现 `# 论文初稿 v1`、`# 最终稿`、`## 正文`、`## 风险与残余说明` 这类系统外壳；正式论文草稿不应保留。
- 题目被放在正文段落里，而不是作为论文题名独立置顶。
- 摘要未稳定覆盖研究背景、目的、方法、发现、意义。
- 关键词可用，但应控制在 3-8 个，使用中文分号分隔，避免过长。
- 正文只有“一、二、三”一级标题，缺少 `1.1`、`1.1.1` 等可写的小标题层级。
- 中间章节缺少“本章小结”，导致章节之间像连续报告，而不是学位论文结构。
- 没有参考文献部件；若 citation map 可用，应保留或生成待核对的参考文献位置。
- 风险提示、readiness、证据边界等系统语言进入稿件外壳；正式草稿应把研究不足自然放入结论末段，审计信息放 sidecar。

## 正式论文外层结构

优先使用学位论文式结构；若任务明确是期刊论文，再使用期刊论文式结构。

### 学位论文式

```text
题名

摘要
关键词
Abstract（如项目要求且有能力可靠生成）
Keywords（如项目要求且有能力可靠生成）
目录（提交包需要时生成；普通 markdown 可省略）

第一章 绪论
1.1 研究背景与问题提出
1.2 研究对象与范围
1.3 研究目的与意义
1.4 国内外研究现状
1.5 研究方法与技术路线
1.6 研究内容与论文框架
本章小结

第二章 文献综述与理论基础 / 概念界定与分析框架
2.1 ...
2.2 ...
本章小结

第三章 至倒数第二章：按 argument tree 展开核心论证
3.1 ...
3.2 ...
本章小结

结论与展望
参考文献
附录（如有）
```

### 期刊论文式

```text
题名
摘要
关键词
Abstract / Keywords（如需要）

一、问题提出
二、研究综述与分析框架
三、核心论证
四、路径或机制分析
五、结语
参考文献
```

不要把两套编号混用。学位论文用“第一章 / 1.1 / 1.1.1”；期刊论文用“一、/ （一）/ 1.”。

## 摘要与关键词

摘要应为 300-500 字，包含五个信息单元：

1. 研究背景：说明问题出现的学术或现实语境。
2. 研究目的：说明本文要解决什么问题。
3. 研究方法：说明采用文献梳理、案例分析、比较研究、机制分析、空间分析等方法中的哪些，必须与实际材料一致。
4. 主要发现：用 2-3 句概括核心结论，不扩写新事实。
5. 研究意义：说明理论或实践价值，语气保守。

关键词 3-8 个，按“研究对象 -> 问题领域 -> 方法/理论工具 -> 应用场景”排序。避免把一句话拆成多个关键词，也不要加入正文没有稳定使用的概念。

## 绪论写法

绪论不要只写背景。它应具备清晰的二级标题：

- `1.1 研究背景与问题提出`：从现实矛盾进入，不写宣传性大背景。
- `1.2 研究对象与范围`：界定对象、时间、空间、材料、案例或研究边界。
- `1.3 研究目的与意义`：分清要解决的问题、理论意义和实践意义。
- `1.4 国内外研究现状`：按议题归纳，不逐篇罗列；最后指出研究缺口。
- `1.5 研究方法与技术路线`：只写真实使用的方法，避免虚构问卷、实测、模拟、访谈或图纸分析。
- `1.6 研究内容与论文框架`：逐章交代每章任务。

绪论末尾可以写“本章小结”，但不重复摘要。

## 中间章节写法

每个中间章节都应有“章题 - 二级标题 - 必要三级标题 - 本章小结”。

章题承担一个论证功能，例如：

- 对象与发展脉络
- 类型划分与特征分析
- 问题诊断与影响因素
- 概念工具与分析框架
- 机制解释与路径组织
- 案例比较与综合评价
- 策略构建与适用条件

二级标题应表达具体分析任务，不要只有“相关分析”“案例研究”“路径探讨”。推荐类型：

- `2.1 概念界定与研究边界`
- `2.2 既有研究的主要议题`
- `2.3 研究缺口与分析框架`
- `3.1 对象类型与特征识别`
- `3.2 主要问题及其成因`
- `3.3 影响机制与约束条件`
- `4.1 分析维度的建立`
- `4.2 关键要素的组织关系`
- `4.3 本章小结`

三级标题只在材料复杂时使用，用于拆分证据组、比较维度、案例维度或机制环节；不要为了显得正式而机械增加三级标题。

## 本章小结

真实论文常用“本章小结”承担章节收束功能。写法要求：

- 100-250 字为宜。
- 回答“本章证明了什么、为下一章提供了什么前提”。
- 不新增材料，不展开下一章内容。
- 不写成“本章首先、其次、最后”的机械目录复述。

## 结论与展望

结论不是摘要重写，也不是策略清单。应包含：

1. 研究结论：按 thesis 和 main argument 回收 2-4 条主要判断。
2. 研究贡献：说明对对象认识、方法框架或实践路径的贡献。
3. 研究不足：自然说明资料、案例、方法或验证范围限制；不要使用 workflow 风险语言。
4. 后续展望：提出可继续验证的问题或材料方向。

不得在结论中首次提出新概念、新案例、新数据或新策略。

## 参考文献

Part 5 `manuscript_v1.md` 应预留参考文献部件，方便 review/citation audit 检查。

- 若 `citation_map.json` 或 `raw-library/metadata.json` 足以生成条目，按 writing-policy 的 GB/T 7714 规则整理。
- 若来源信息不足，不得伪造页码、卷期、出版社或作者；保留“参考文献（待核对）”并把缺失项交给 review/report，而不是写进正文。
- 正文中不要出现 `source_id`、artifact 名、risk level 或内部 gate 名称。

## 禁止写法

- `# 论文初稿 v1`、`# 最终稿`、`## 正文`、`## 风险与残余说明` 这类系统标题不得进入正式论文草稿。
- 不得把 “Part 5 / Part 6 / canonical / handoff / readiness / gate / artifact” 写进正文。
- 不得写“已核验资料显示”“证据链表明”“可回溯资料显示”等系统审计语言。
- 不得用“本文进一步指出”机械串联所有段落；只在真正推进论证时使用。
- 不得把参考论文结构中的具体主题词替换到当前论文。
- 不得为了补齐正式结构而虚构研究方法、英文摘要、参考文献条目或 citation 信息。

## 输出自检

交付 `manuscript_v1.md` 前检查：

- 题名是否独立置顶，而不是藏在“正文”里？
- 是否删除了 workflow 外壳和风险说明外壳？
- 摘要是否覆盖背景、目的、方法、发现、意义？
- 关键词是否 3-8 个且与正文核心概念一致？
- 正文编号体系是否统一？
- 绪论是否具备对象、范围、目的、意义、现状、方法、框架？
- 中间章节是否有可写的二级标题，而不是大段平铺？
- 每个核心章节是否有“本章小结”或等价收束？
- 结论是否回收论证而不是新增判断？
- 参考文献是否存在，且没有伪造缺失信息？
