# AGENTS.md — Local Web Console

## Purpose
- `local-web/` contains the local browser console for operating the research-to-manuscript workflow.
- It is an interface layer only: it may read workflow state, display artifacts, and call approved CLI/runtime actions.
- It must not become a research evidence layer and must not store paper sources, wiki pages, manuscript drafts, or writing-policy material.

## Directory Contract

```
local-web/server.py          # stdlib local HTTP server and allowlisted API actions
local-web/static/index.html  # single-page browser console
local-web/static/styles.css  # UI styling
local-web/static/app.js      # client-side status rendering and action calls
local-web/tests/             # tests for the local web console
```

## Operation Rules
- The server must default to `127.0.0.1` and must not expose arbitrary shell execution.
- API actions must be allowlisted and translated to existing `cli.py` commands or approved runtime scripts.
- Human gates may be confirmed only through explicit user-submitted notes or candidate selections.
- The UI must clearly distinguish root project state from isolated workspace state.
- Part 6 must remain opt-in and must never be launched automatically by a previous stage action.

## Naming Rules
- Python files use `snake_case.py`.
- Frontend files use lowercase hyphen-free names already declared above.
- API action IDs use lowercase hyphen-case and must be documented in `server.py`.

## Cleanup Rules
- Runtime logs generated by the web console stay in memory for the current server session unless explicitly exported later.
- Do not create persistent UI cache files under workflow artifact directories.
- Temporary test screenshots or reports must stay outside canonical workflow directories.
